# OOP3

1. 프로젝트 개요  
본 팀 프로젝트는 한 학기 동안 학습한 객체지향 프로그래밍의 핵심 개념을 종합적으로 활용하여, 4인 1팀이 협업하여 자유 주제의 C# 콘솔 애플리케이션을 설계하고 구현하는 것을 목표로 한다.  
단순히 동작하는 프로그램을 만드는 것이 아니라, 클래스 설계 · 상속과 다형성 · 인터페이스 · 예외 처리 · 델리게이트/람다 등 객체지향의 본질이 코드에 자연스럽게 드러나는 결과물을 만들어야 한다. 

< 학습 목표 >  
• 실제 프로젝트 수준에서 클래스 설계와 객체지향 원칙을 직접 적용한다.  
• 상속, 다형성, 인터페이스, 추상화를 적절히 사용하여 확장 가능한 구조를 설계한다.  
• 예외 처리, 델리게이트, 람다를 활용해 견고하고 유연한 코드를 작성한다.  
• 팀원 간 역할 분담과 코드 통합을 통해 협업 경험을 쌓는다. 

2. 주제 선정  
주제는 자유롭게 선정한다. 단, 다양한 객체(엔티티)가 등장하고 그 사이에 상속·구현 관계가 자연스럽게 만들어질 수 있는 도메인을 권장한다.  
※ 같은 도메인을 여러 팀이 선택해도 무방하나, 구현은 각 팀이 독자적으로 수행 필수.

주제 예시 (참고용)  
• 도서관 / 서점 관리 시스템 (책, 회원, 대출, 사서)  
• 카페 / 식당 주문 시스템 (메뉴, 주문, 직원, 결제)  
• RPG 게임 시뮬레이터 (캐릭터, 몬스터, 아이템, 스킬)  
• 영화관 예매 시스템 (영화, 상영관, 좌석, 예매)  

3. 필수 기술 요구 사항  
| 항목 | 과제 최소 수량 | 적용 내용 |
|---|---|---|
| 일반 클래스 | 10개 이상 | User, Session, AnalysisResult 등 클래스 설계 |
| 구조체(struct) | 1개 이상 | Position 구조체 구현 |
| 메서드 | 30개 이상 | 기능별 메서드 구현 |
| 메서드 오버로딩 | 3쌍 이상 | 같은 이름의 메서드 다중 정의 |
| out 키워드 | 2개 이상 | 다중 반환 메서드 구현 |
| 상속 계층 | 2개 이상 | Character → Player / Monster 구조 |
| 메서드 오버라이딩 | 5개 이상 | virtual / override 기반 재정의 |
| base 키워드 | 3회 이상 | 부모 생성자 및 메서드 호출 |
| protected 멤버 | 필수 | 상속 클래스 접근 제한 적용 |
| 추상 클래스 | 1개 이상 | abstract Entity 클래스 구현 |
| 인터페이스 | 3개 이상 | IAttackable 등 인터페이스 구현 |
| 표준 인터페이스 | 1개 이상 | IComparable 구현 |
| 다중 인터페이스 상속 | 1개 이상 | 한 클래스에서 다중 인터페이스 구현 |
| 제네릭 | 1개 이상 | Generic Repository 구현 |
| 인덱서 | 1개 이상 | [ ] 접근 인덱서 구현 |
| try-catch-finally | 5회 이상 | 입력 검증 및 예외 처리 |
| 사용자 정의 예외 | 1개 이상 | InvalidInputException 구현 |
| throw 사용 | 필수 | 잘못된 입력 예외 발생 |
| 델리게이트 | 1개 이상 | 이벤트 콜백 처리 |
| 람다식 | 5회 이상 | 정렬 및 필터 처리 |
| 컬렉션 활용 | 필수 | List<T>, Dictionary<K,V> 사용 |
 
< 정성적 요구사항 >  
• 단순히 개수만 채우지 말고, 각 개념이 도메인 안에서 의미 있게 사용되어야 함.  
• 클래스명·메서드명·변수명은 일관된 네이밍 규칙(Pascal/camel case)을 따른다.  
• 한 메서드가 100줄을 넘지 않도록 적절히 분리한다.  
• 필드는 가능한 private으로, 외부 접근은 속성(Property)을 통해서 한다 (캡슐화).  
• 의미 없는 빈 catch { } (예외 무시) 는 감점 대상이다. 

4. 금지 사항  
• 외부 라이브러리사용 금지. .NET 표준 라이브러리만 사용한다.  
• 팀 간 코드 공유 및 복사 금지.  
• AI(Copilot, ChatGPT 등) 생성 코드를 그대로 붙여넣기 금지. 참고는 가능하나, 모든 코드는 본인이 이해하고 설명할 수 있어야 한다 (시연 시 무작위 질문).
